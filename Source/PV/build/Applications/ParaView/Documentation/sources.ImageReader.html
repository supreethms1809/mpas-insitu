Image Reader (ImageReader)

Reads raw regular rectilinear grid data from a file. The dimensions and type of the data must be specified.
The Image reader reads raw, regular, rectilinear grid (image/volume) data from a file. Because no metadata is provided, the user must specify information about the size, spacing, dimensionality, etc. about the dataset.
Property Description Default(s) Restrictions
FilePrefix The text string contained in this property specifies the file prefix (directory plus common initial part of file name) for the raw binary uniform rectilinear grid dataset.

The value(s) must be a filename (or filenames).
FilePattern The text string contained in the property specifies the format string to determine the file names necessary for reading this dataset. In creating the filenames, %s will be replaced by the prefix and %d by a digit which represents the slice number in Z. The format string is the same as that used by printf. %s
DataScalarType The value of this property indicates the scalar type of the pixels/voxels in the file(s): short, int, float ... 4

The value(s) is an enumeration of the following:

  • char (2)
  • unsigned char (3)
  • short (4)
  • unsigned short (5)
  • int (6)
  • unsigned int (7)
  • long (8)
  • unsigned long (9)
  • float (10)
  • double (11)
DataByteOrder This property indicates the byte order of the binary file(s). 0

The value(s) is an enumeration of the following:

  • BigEndian (0)
  • LittleEndian (1)
FileDimensionality This property indicates whether the file(s) in this dataset contain slices (2D) or volumes (3D). 3

The value(s) is an enumeration of the following:

  • 2 (2)
  • 3 (3)
DataOrigin The coordinate contained in this property specifies the position of the point with index (0,0,0). 0.0 0.0 0.0
DataSpacing This property specifies the size of a voxel in each dimension. 1.0 1.0 1.0
DataExtent This property specifies the minimum and maximum index values of the data in each dimension (xmin, xmax, ymin, ymax, zmin, zmax). 0 0 0 0 0 0
NumberOfScalarComponents This property specifies the number of components the scalar value at each pixel or voxel has (e.g., RGB - 3 scalar components). 1
ScalarArrayName This property contains a text string listing a name to assign to the point-centered data array read. ImageFile
FileLowerLeft This property determines whether the data originates in the lower left corner (on) or the upper left corner (off). Most scientific data is written with a right-handed axes that originates in the lower left corner. However, several 2D image file formats write the image from the upper left corner. 1

Accepts boolean values (0 or 1).
